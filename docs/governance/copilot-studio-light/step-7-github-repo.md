# Étape 7 — Repo GitHub : Modèle d'agent & documentation

Description :
- Pour les agents approuvés, un repository ou un dossier modèle est créé/ mis à jour sur GitHub contenant la fiche agent, README, instructions opérationnelles, CHANGELOG et les fichiers de configuration nécessaires.

Contenu recommandé du repo modèle :
- README.md (description, owner, contact)
- docs/ (instructions détaillées, playbooks)
- agent.zip (ou lien vers SharePoint)
- CHANGELOG.md
- governance.md (métadonnées, classification)

Bonnes pratiques :
- Protéger les branches principales, utiliser PRs pour modifications importantes.
- Lier le README à la fiche dans SharePoint et aux tickets d'approbation.
