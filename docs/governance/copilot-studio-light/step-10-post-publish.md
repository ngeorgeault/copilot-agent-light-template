# Étape 10 — Post-publication : Mise à jour & suivi

Description :
- Après publication, mettre à jour la fiche agent dans SharePoint, notifier le PO et suivre l'usage via télémétrie. Mettre en place des revues périodiques et un processus de retrait en cas d'incident.

Actions :
- Mettre à jour la liste SharePoint (Statut, Lien publication, Date dernière revue).
- Planifier une revue périodique (ex. 6 mois) et contrôler les logs d'utilisation.
- Procédure de retrait : retirer l'application via admin M365, informer les utilisateurs, garder un historique des versions.
